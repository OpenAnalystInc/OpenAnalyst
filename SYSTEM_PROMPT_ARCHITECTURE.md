# System Prompt Architecture Documentation

## Overview
This document provides a comprehensive breakdown of the system prompt generation architecture in the Open Analyst codebase. The system prompt is constructed from multiple components, each responsible for specific sections.

## System Prompt Structure

The system prompt is generated by the `SYSTEM_PROMPT` function in `src/core/prompts/system.ts` and follows this structure:

### 1. ROLE DEFINITION
**Source**: `getModeSelection()` function in `src/shared/modes.ts`
**Component**: Mode configuration system
**Description**: Defines the AI's role based on the selected mode (code, architect, ask, etc.)
**Location in Code**: 
```typescript
const { roleDefinition, baseInstructions } = getModeSelection(mode, promptComponent, customModeConfigs)
```

### 2. MARKDOWN RULES
**Source**: `markdownFormattingSection()` in `src/core/prompts/sections/markdown-formatting.ts`
**Component**: Markdown formatting rules
**Description**: Defines how to format code references and file links in responses
**Content**: Rules for making language constructs and filenames clickable with line numbers

### 3. TOOL USE
**Source**: `getSharedToolUseSection()` in `src/core/prompts/sections/tool-use.ts`
**Component**: Tool usage guidelines
**Description**: Explains how to use tools with XML-style formatting
**Content**: Instructions for tool parameter formatting and usage patterns

### 4. TOOL DESCRIPTIONS
**Source**: `getToolDescriptionsForMode()` in `src/core/prompts/tools/index.ts`
**Component**: Dynamic tool generation system
**Description**: Generates descriptions for all available tools based on mode configuration
**Dependencies**:
- Mode groups (read, edit, browser, command, mcp, modes)
- Tool availability based on experiments
- CodeIndexManager status
- MCP server availability

**Tool Groups** (from `src/shared/tools.ts`):
- **read**: read_file, fetch_instructions, search_files, list_files, list_code_definition_names, codebase_search
- **edit**: apply_diff, edit_file, write_to_file, insert_content, search_and_replace, new_rule
- **browser**: browser_action
- **command**: execute_command
- **mcp**: use_mcp_tool, access_mcp_resource
- **modes**: switch_mode, new_task

### 5. TOOL USE GUIDELINES
**Source**: `getToolUseGuidelinesSection()` in `src/core/prompts/sections/tool-use-guidelines.ts`
**Component**: Tool usage best practices
**Description**: Provides guidelines for effective tool usage and workflow
**Content**: Best practices for tool selection and usage patterns

### 6. MORPH INSTRUCTIONS (Experimental)
**Source**: `getMorphInstructions()` in `src/core/prompts/tools/edit-file.ts`
**Component**: Morph FastApply experiment
**Description**: Instructions for using Morph's edit_file tool when enabled
**Condition**: Only included when `experiments.morphFastApply === true`

### 7. MCP SERVERS SECTION
**Source**: `getMcpServersSection()` in `src/core/prompts/sections/mcp-servers.ts`
**Component**: MCP (Model Context Protocol) integration
**Description**: Lists available MCP servers and their capabilities
**Condition**: Only included when mode has MCP group and MCP servers are available

### 8. CAPABILITIES
**Source**: `getCapabilitiesSection()` in `src/core/prompts/sections/capabilities.ts`
**Component**: System capabilities description
**Description**: Describes what the AI can do with available tools
**Content**: 
- File operations capabilities
- Code search capabilities
- Browser interaction (if enabled)
- MCP server capabilities (if available)
- Codebase search capabilities (if enabled)

### 9. MODES
**Source**: `getModesSection()` in `src/core/prompts/sections/modes.ts`
**Component**: Mode management system
**Description**: Lists all available modes with descriptions
**Content**: 
- Built-in modes (code, architect, ask, etc.)
- Custom modes
- Instructions for creating new modes

### 10. RULES
**Source**: `getRulesSection()` in `src/core/prompts/sections/rules.ts`
**Component**: System rules and constraints
**Description**: Defines operational rules and constraints
**Content**:
- File path handling rules
- Directory restrictions
- Codebase search requirements
- File search guidelines
- Project organization rules

### 11. SYSTEM INFORMATION
**Source**: `getSystemInfoSection()` in `src/core/prompts/sections/system-info.ts`
**Component**: Environment information
**Description**: Provides system environment details
**Content**:
- Operating system information
- Shell information
- Home directory
- Current workspace directory
- File structure overview

### 12. OBJECTIVE
**Source**: `getObjectiveSection()` in `src/core/prompts/sections/objective.ts`
**Component**: Task execution methodology
**Description**: Defines the AI's approach to task execution
**Content**:
- Iterative task breakdown
- Goal prioritization
- Tool usage methodology
- Completion requirements

### 13. CUSTOM INSTRUCTIONS
**Source**: `addCustomInstructions()` in `src/core/prompts/sections/custom-instructions.ts`
**Component**: Custom instruction management
**Description**: Combines various custom instructions and rules
**Sources**:
- Global custom instructions
- Mode-specific instructions
- Local rule files (.oacode/rules-*.md)
- Global rule files
- AGENTS.md file (if enabled)
- Roo ignore instructions
- Rule toggle states

## File-Based Custom System Prompts

When a custom system prompt file exists (`.oacode/system-prompt.md`), the system uses a simplified structure:

1. **Role Definition** (from mode selection)
2. **Custom System Prompt** (from file)
3. **Morph Instructions** (if enabled)
4. **Custom Instructions** (combined from various sources)

## Key Components and Their Responsibilities

### Mode System (`src/shared/modes.ts`)
- Manages mode configurations
- Handles tool group assignments
- Provides mode selection logic
- Manages custom mode overrides

### Tool System (`src/core/prompts/tools/`)
- Generates tool descriptions
- Manages tool availability based on mode
- Handles experimental features
- Provides tool-specific instructions

### Sections System (`src/core/prompts/sections/`)
- Modular prompt section generation
- Environment-specific content
- Dynamic content based on configuration
- Localization support

### Custom Instructions System (`src/core/prompts/sections/custom-instructions.ts`)
- Loads and combines multiple instruction sources
- Handles rule file management
- Manages toggle states
- Provides fallback mechanisms

## Configuration Dependencies

The system prompt generation depends on:

1. **Mode Configuration**: Determines role, tools, and behavior
2. **Experiments**: Enables/disables experimental features
3. **MCP Configuration**: Provides additional tool capabilities
4. **CodeIndexManager**: Enables semantic code search
5. **File System**: Loads custom instructions and rules
6. **Extension Context**: Provides workspace and global state
7. **API Configuration**: Determines available capabilities

## Dynamic Content Generation

The system prompt is highly dynamic, with content generated based on:

- **Current Mode**: Affects role definition and available tools
- **Available Services**: MCP servers, code indexing, browser support
- **User Configuration**: Custom instructions, rule toggles, experiments
- **Workspace State**: Local rules, project-specific settings
- **Environment**: Operating system, shell, file structure

This architecture ensures that the system prompt is always tailored to the current context while maintaining consistency across different environments and configurations.
